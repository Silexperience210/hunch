### Task 2 — Generate project Nostr pseudonym keypair

The HUNCH_NSEC secret key for the project pseudonym MUST be generated by the user under their own opsec. **Not auto-generated in a Claude session** because:

1. Any nsec emitted into the chat history is potentially logged / compromised
2. The project pseudonym key needs hardware-backed storage (Ledger / Trezor / Nostr.Hardware / air-gapped device)
3. Per CONTEXT.md decision D-05, full pseudonymity is load-bearing — the key's provenance should be controlled by the user, not by an AI agent's network call

**User action:**
```bash
# Option 1: use rust-nostr nostr-cli (recommended for opsec)
nostr-cli key generate --bech32

# Option 2: NDK / nostr-tools quickly (less opsec)
bun -e 'import("nostr-tools").then(m => { const sk = m.generateSecretKey(); console.log("nsec:", m.nip19.nsecEncode(sk)); console.log("npub:", m.nip19.npubEncode(m.getPublicKey(sk))); })'

# Then store nsec in .env.local (gitignored already):
echo "HUNCH_NSEC=nsec1..." >> .env.local
```

The npub should be backfilled into:
- `CONTRIBUTING.md` Security section (replace `npub-TBD`)
- `CODE_OF_CONDUCT.md` Enforcement section (replace `npub-TBD`)
- `docs/audit-shortlist-outreach.md` outreach template (replace `npub-TBD`)
